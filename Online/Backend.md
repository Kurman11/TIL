## API(개발자의 암호문)
* 클라이언트와 서버를 이어주는 개념이다.

* 클라이언트의 관점
  * CREATE(올려줘) / POST
  * READ(불러와줘) / GET
  * UPDATE(바꿔줘) / PUT(전체)/PATCH(일부)
  * DELETE(지워줘) / DELETE

* 서버에서 클라이언트로
  * 200 잘 됐어
  * 400 (요청 에러) : 클라이언트에서 잘 안됐어 (ex 403(서버가 요청을 거부하고있다.)에러 404(서버가 요청한 페이지를 찾을 수 없다.)에러)
  * 500 (서버 오류) : 서버쪽에서 잘 안됐어

          * API 가운데 데이터를 주고받는 형식 -> JSON
