# branch
> 독립적으로 어떤 작업을 진행하기 위한 개념이다.
필요에 의해 만들어지는 각각의 브랜치는 다른 브랜치의 영향을 받지 않기 때문에, 여러 작업을 동시에 진행할 수 있다.

```bash
$ git branch <이름>
#브랜치 생성
```
```bash
$ git checkout <이름>
# 브래치 이동
```
```bash
$ git checkout -b <이름>
# 브랜치 생성 및 이동
# 더 많이 사용한다 편함
```
```bash
$ git branch -d <이름>
# 브랜치 삭제
```
```bash
$ git branch
# 브랜치 목록
```
```bash
$ git merge <이름>
# 브랜치 병합
```
